一、项目定位

mattpocock/skills 是知名 TypeScript 教育者 Matt Pocock 从自己的 .claude 目录里直接抽出来开源的一套 Claude Code 技能包（Agent Skills）。 它不是一个 Agent 框架，也不涉及 MCP 协议 / Zod schema——它的语言构成是 100% Shell + Markdown，本质是"把资深工程师的方法论翻译成 Agent 能消费的 SKILL.md"。

副标题很坦诚："Skills for Real Engineers. Straight from my .claude directory." —— 面向严肃软件工程，而非快速原型。
事实校准： 你的任务描述提到 "mcp-tool / MCP 协议 / Zod schema validation / TypeScript 类型系统驱动 tool calling"——这些不属于本仓库。 本仓库是 Claude Code 的 SKILL.md（YAML frontmatter + Markdown 正文）技能集合，靠提示词工程而非类型系统工作。本报告分析真实仓库。

三、SKILL.md 规范深度解析

Claude Code 的每个技能都是一个文件夹，核心是一个 SKILL.md，由两部分组成：

  1. YAML frontmatter（--- 之间）——告诉 Claude 何时该用这个技能；
  2. Markdown 正文——技能触发后 Claude 遵循的指令、示例、规范。

关键 frontmatter 字段

字段含义
name必须与父文件夹名完全一致
description最关键字段——Claude 靠它判断"什么场景调用此技能"，要写得具体
disable-model-invocation设为 true 则模型不会自动调用，只能由用户显式 /技能名 触发
allowed-tools（可选）限制此技能可用的工具

四、真实 SKILL.md 示例（verbatim）

以下是 skills/engineering/setup-matt-pocock-skills/SKILL.md 的真实 frontmatter：

name: setup-matt-pocock-skills
description: Sets up an `## Agent skills` block in AGENTS.md/CLAUDE.md and `docs/agents/`
  so the engineering skills know this repo's issue tracker (GitHub or local markdown),
  triage label vocabulary, and domain doc layout. Run before first use of `to-issues`,
  `to-prd`, `triage`, `diagnose`, `tdd`, `improve-codebase-architecture`, or `zoom-out`
  — or if those skills appear to be missing context about the issue tracker, triage
  labels, or domain docs.
disable-model-invocation: true

正文描述了一个三阶段工作流：

阶段做什么
Exploration 探索审计 repo 现状：git remotes、现有 AGENTS.md/CLAUDE.md、context 文档、ADR 目录、.scratch/ 文件夹
Decision 决策引导用户做 3 个配置：① issue tracker 位置 ② triage 标签词汇 ③ 领域文档布局——每个都给默认推荐
Implementation 落地生成配置文件：更新 CLAUDE.md/AGENTS.md，再加 3 个 md 文件描述 issue tracker、标签映射、领域文档结构
注意 disable-model-invocation: true 的用意：这是个"一次性配置"技能，不该被模型在日常对话里随意触发，只能用户主动 /setup-matt-pocock-skills 调用。

六、推荐工作流

这些技能不是孤立的，作者设计了一条"从需求到交付"的链路：

flowchart LR Start(["需求"]) --> A["/grill-with-docs
追问澄清+更新ADR"] A --> B["/to-prd
会话→正式PRD"] B --> C["/to-issues
拆成可独立认领的垂直切片"] C --> D["/tdd
红-绿-重构"] D --> E{"发现 Bug?"} E -->|是| F["/diagnose
复现→最小化→假设→注入→修复"] F --> D E -->|否| G["/improve-codebase-architecture
治理设计退化"] G --> H["/handoff
会话压缩, Agent 间交接"] H --> Finish(["交付"])

配套的 caveman（穴居人模式，压缩到极简表达）和 zoom-out（要求 Agent 先建立更广视野）则是贯穿全程的"调节器"。

七、CLAUDE.md / AGENTS.md / CONTEXT.md 三件套

文件角色
CLAUDE.md / AGENTS.mdAgent 配置入口，setup-matt-pocock-skills 会在其中写入 ## Agent skills 块，记录本 repo 的 issue tracker、triage 标签、领域文档布局
CONTEXT.md项目"共同语言"文档——一次性建立领域术语，之后每次会话用极少 token 沟通。作者称之为"投资一次，受益每次"
docs/adr/架构决策记录，grill-with-docs 会内联更新它
这套三件套的本质：把"项目知识"从会话上下文（易失、烧 token）外置成仓库文件（持久、可 review、可 git 管理）——与 AgentMemory 把记忆外置成 SQLite、OpenHuman 外置成 Memory Tree 是同一个时代命题的不同解法。

八、安装与上手

# 30 秒安装
npx skills@latest add mattpocock/skills

# 安装器交互流程:
#  1. 选择要装哪些技能 + 指定目标 AI agent
#  2. 自动跑 /setup-matt-pocock-skills:
#       - 选 issue tracker (GitHub / Linear / 本地 markdown)
#       - 定义 triage 标签词汇
#       - 指定领域文档目录
#  3. 把技能加载进 .claude 目录

# 之后在 Claude Code 里作为斜杠命令调用:
#   /diagnose   /tdd   /caveman   /grill-with-docs  ...

技能与任意模型/Agent 兼容，但一等公民是 Claude Code。.claude-plugin/ 目录让它能作为 Claude Code 插件被发现。

十、适用场景与优劣势

适合

  • 重度使用 Claude Code、想把"野生 AI 流程"标准化的工程师
  • 从原型走向生产的团队——需要纪律性
  • Tech Lead 想让团队 Agent 行为对齐工程文化
  • TypeScript / 全栈生态（作者的母语圈）

局限

  • 纯提示词，无强制性——Claude 可能不严格遵循
  • 偏 Matt Pocock 个人工作流，未必契合所有团队
  • 100% Shell/Markdown，无代码逻辑，本质是"高质量约定"而非"工具"
  • 效果强依赖底层模型能力
