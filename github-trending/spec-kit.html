Spec Kit 是 GitHub 官方推出的规格驱动开发（SDD）工具包：用 specify CLI 创建项目，再以 7 条斜杠命令把"宪章 → 规格 → 澄清 → 方案 → 任务 → 实施"串成完整闭环。支持 30+ AI 编码 Agent、120+ 社区扩展，已被 Claude Code 与 GitHub Copilot 官方文档列为"工程化使用方式"。star 数逼近 10 万，是 AI 编程时代的"Maven / npm 工程约定"候选。

01项目概述

仓库 github/spec-kit 来自 GitHub 官方，2026 年 5 月单日 +1,299 stars，总数逼近 10 万。它要解决的是 AI 编程时代一个具体的矛盾——越强的 LLM，越需要更结构化的输入。

传统开发把规格当作可丢弃的草稿；SDD 反转了这个关系——规格是可执行的源头工件，代码反而是它的"衍生物"。

02方法论：规格驱动开发（SDD）

SDD 的四条主张可以浓缩成一句话："先回答 What，再决定 How"——并把每一步沉淀为可被 AI 消费的工件。

PRINCIPLE 1
Intent-Driven
→ 意图驱动开发
先回答 What，再决定 How。
PRINCIPLE 2
Rich Specs + Guardrails
→ 富规格 + 组织护栏
规格内含约束、合规与组织偏好。
PRINCIPLE 3
多步精炼
→ 优于一次性生成
让 AI 在多步反馈中收敛而非"一气呵成"。

第四条原则是技术性的：充分利用先进 AI 模型解释规格的能力——规格越富、Agent 越强，闭环越完整。

7 条核心命令

命令作用
/speckit.constitution制定项目治理原则
/speckit.specify定义需求和用户故事
/speckit.plan制定技术实施方案
/speckit.tasks生成可执行任务清单
/speckit.implement按任务清单实施
/speckit.taskstoissues把任务转成 GitHub Issues
/speckit.clarify · analyze · checklist可选增强（澄清、一致性分析、QA 清单）

04工件 + 目录约定

specify init 后会落下一个标准目录结构——这是 Spec Kit 试图成为"AI 编程时代的 Maven / npm 工程约定"的关键设计。

.
├── .specify/
│   ├── memory/constitution.md
│   ├── scripts/   # 检查依赖、创建 feature、setup-plan
│   ├── templates/ # spec / plan / tasks 模板
│   └── extensions/
├── specs/
│   └── [feature-name]/
│       ├── spec.md          # 需求与故事
│       ├── plan.md          # 技术方案
│       ├── tasks.md         # 任务清单
│       ├── data-model.md
│       ├── contracts/       # API / 协议
│       ├── research.md
│       └── quickstart.md
└── .claude/commands/        # Claude Code 自定义命令

模板解析的 4 层优先级

  1. 项目本地覆盖 .specify/templates/overrides/ — 最高优先级
  2. Presets .specify/presets/templates/
  3. Extensions .specify/extensions/templates/
  4. 核心模板 .specify/templates/ — 最低
NOTE · 工件类型
Constitution（治理原则）· Spec（功能性需求 + 用户故事 + 验收条件）· Plan（技术架构、栈选型、实施策略）· Tasks（带依赖、可并行的任务列表）· Contracts（API 规格、数据模型）· Research（库选型、技术调研记录）。

06三大典型场景

GREENFIELD
0→1 开发
→ 从高层需求一路到 MVP
用 SDD 闭环从想法生成可工作的应用。
EXPLORATION
创意探索
→ 并行尝试不同技术栈 / UX
规格不变，Plan / Tasks 可多次重算。
BROWNFIELD
迭代增强
→ 遗留系统增量改造
把 SDD 用于既有代码库的现代化。

适用人群

  • 企业团队. 需要在合规、可追溯、可审计的约束下使用 AI 编码。
  • 技术 Lead. 希望让团队的 AI 协作有统一流程而非各自 prompt。
  • 创业团队. 0→1 从想法到 MVP 的高速通道。
  • 遗留系统改造者. 把 SDD 用于 brownfield 增量改造。

08如何快速上手

持久化安装（推荐）

# 推荐使用 uv
uv tool install specify-cli --from git+https://github.com/github/spec-kit.git@vX.Y.Z

# 或使用 pipx
pipx install git+https://github.com/github/spec-kit.git

一次性运行（不安装）

uvx --from git+https://github.com/github/spec-kit.git specify init my-project

初始化 + 运行典型工作流

specify init my-awesome-app
cd my-awesome-app

# 然后在 Claude Code 中：
/speckit.constitution      # 1. 制定项目宪章
/speckit.specify           # 2. 编写规格
/speckit.clarify           # 3. 澄清歧义
/speckit.plan              # 4. 技术方案
/speckit.tasks             # 5. 拆任务
/speckit.implement         # 6. 实施
/speckit.taskstoissues     # 7.（可选）转为 GitHub Issues

安装扩展与 Preset

specify extension search
specify extension add architecture-guard

specify preset search
specify preset add security-first

查看支持的 Agent

specify integration list
