一、项目定位

AgentMemory（仓库 rohitg00/agentmemory）是一套面向 AI 编程 Agent 的持久化记忆基础设施。 它的口号直白："Your coding agent remembers everything. No more re-explaining."——让 Claude Code、Cursor、Codex 等 Agent 跨会话自动记住代码库架构、Bug 上下文、用户偏好。

它在 LongMemEval-S 基准上拿到 95.2% Recall@5，显著领先 mem0（68.5%）与 Letta（83.2%）；同时号称零外部数据库依赖、年 token 成本约 $10。

定位关键词：Hook 驱动的自动捕获 + 4 层记忆模型 + 三流融合检索（BM25 + 向量 + 知识图谱 + RRF） + iii engine 统一运行时。

二、核心架构总览

AgentMemory 整体跑在 iii engine 之上——一个把 Express / SQLite / Redis / pm2 / Prometheus 统一成单一运行时原语的引擎。每个 function、trigger、state scope、stream 都是 iii primitive，可观测性"白送"。

设计哲学： "fire-and-forget" 摄入——PostToolUse hook 静默捕获，后台 cron trigger 做 consolidation；多 Agent 协作用 iii 内建的 leases/signals 而非轮询。

三、iii engine 与 iii-config.yaml 深度解析

整个系统的运行时定义在 iii-config.yaml 里，声明了 7 个 worker：

Worker职责关键配置
iii-httpHTTP 服务器:3111 · localhost · CORS（GET/POST/PUT/DELETE/OPTIONS）
iii-state状态管理文件型 KV · ./data/state_store.db
iii-queue队列适配器builtin 实现
iii-pubsub发布/订阅local 实现
iii-cron定时调度KV adapter（驱动后台 consolidation）
iii-stream流服务器:3112 · 文件型 ./data/stream_store
iii-exec执行 workerwatch src/**/*.ts → run node dist/index.mjs

可观测性配置：memory exporter、service name agentmemory、采样率 1.0、metrics + console logging 双开。

关键判断：为什么"零外部数据库"成立

state 与 stream 都用文件型 KV持久化（state_store.db / stream_store），向量索引在内存中——所以不需要 Qdrant / pgvector / Redis。 代价是单机为主：要做多实例 fan-out 才需要额外加 iii-pubsub 等 worker。

四、12 Hook 自动捕获管线

AgentMemory 最核心的"无感"体验来自 Hook。以 PostToolUse Hook 为例，完整管线是：

hook 触发"] --> dedup["SHA-256 去重
5 分钟窗口"] dedup --> priv["隐私过滤
剥离 secrets/API key"] priv --> raw[("存储原始观察")] raw --> compress["LLM 压缩
→结构化 facts+concepts+narrative"] compress --> embed["向量 embedding
6 provider + 本地"] embed --> index[("索引: BM25 + 向量")]

而 SessionStart / SessionEnd Hook 负责"开局注入上下文"：

flowchart LR start["SessionStart
hook 触发"] --> profile["加载项目 profile
top concepts/files/patterns"] profile --> hybrid["混合检索
BM25+向量+图谱"] hybrid --> budget["Token 预算
默认 2000 tokens"] budget --> inject["注入对话上下文"]
设计要点：捕获是同步轻量 + 异步重活——hook 同步只做去重+过滤+存原始，重的 LLM 压缩与 embedding 走后台。这与 OpenHuman Memory Tree 的"热路径/后台"哲学异曲同工。

五、4 层记忆模型

记忆按"仿人类认知"分 4 层，由后台 consolidation 逐层提炼：

层级内容来源
Working 工作记忆原始观察（raw observations）Hook 直接捕获
Episodic 情节记忆会话摘要（session summaries）SessionEnd 时聚合
Semantic 语义记忆抽取的事实（extracted facts）cron consolidation 提炼
Procedural 程序性记忆工作流（workflows / "怎么做"）跨会话模式归纳

每层都会被版本化、可回放；viewer（:3113）能逐事件渲染并带时间轴拖动 + 倍速。

六、三流融合检索与 RRF

检索是 AgentMemory 拿下 95.2% Recall 的关键。它同时跑三路再融合：

检索流机制擅长
BM25关键词匹配 + 同义词扩展精确术语、专有名词
向量搜索余弦相似度 · 6 provider + 本地 all-MiniLM-L6-v2语义近似、改写表述
知识图谱实体匹配 + BFS 遍历关系链、间接关联

RRF 融合细节

三路结果用 Reciprocal Rank Fusion（RRF，k=60）融合，并做会话多样化（每个 session 最多 3 条结果），避免单一会话刷屏。

RRF score(d) = Σ_streams  1 / (k + rank_stream(d))      # k = 60

# memory_smart_search 的瀑布执行:
#   BM25 scan → embedding lookup → RRF fusion → reranker
# 三路并行, 融合后 reranker 收尾, 可在 viewer :3113 看到 waterfall
为什么 RRF 而不是加权求和： RRF 只依赖排名不依赖各路分数量纲，BM25 分数和余弦相似度根本不可比——RRF 用 1/(k+rank) 巧妙绕开了归一化难题。k=60 是社区经验值，抑制长尾噪声。

七、API 与 MCP 工具面

MCP Server（packages/mcp/）

  • 51 个工具、6 resources、3 prompts、4 skills
  • 核心 8 个：memory_recall / memory_save / memory_smart_search / memory_sessions / memory_patterns / memory_compress_file / memory_profile / memory_file_history
  • 扩展 43 个：图查询、consolidation、团队共享、审计、Git 快照、多 Agent 租约、P2P 同步…

REST API（src/triggers/api.ts）

  • 107 个端点 @ :3111
  • POST /agentmemory/smart-search
  • POST /agentmemory/observe
  • POST /agentmemory/remember
  • POST /agentmemory/session/start
  • GET /agentmemory/health（始终公开）
  • 受保护端点用 AGENTMEMORY_SECRET 做 Bearer 认证

顶层目录结构

src/核心记忆引擎 + 检索逻辑 + triggers/api.ts
packages/mcp/MCP 服务器实现
integrations/各 Agent 插件（claude code / codex / openclaw / hermes / pi）
benchmark/LongMemEval 等基准与对比
test/~800 测试
已知坑（Issue #159）： 独立运行的 MCP 工具不会代理到正在运行的 agentmemory server——MCP 的读写与 hook 捕获的观察是隔离的。多入口部署时要留意一致性。

优势

  • Hook 自动捕获是真正的"无感"，竞品多需手动
  • 三流 RRF 检索在基准上领先明显
  • 零外部 DB，部署极简
  • 真·多 Agent（MCP+REST 双暴露 + 租约/信号）

风险/短板

  • 深度绑定 iii engine——这是个相对小众的运行时，生态与长期维护是变量
  • 单机为主，水平扩展需自行加 worker
  • MCP/server 隔离问题（#159）说明多入口一致性仍在打磨
  • 95.2% 来自单一基准 LongMemEval-S，泛化性待更多验证

十、典型使用场景

  • 个人开发者：让 Claude Code 永久记住"本项目用 pnpm workspace + Turbo"，不再每次会话重复交代。
  • 多 Agent Debug：多个 Agent 协作排查长链路问题，共享同一份 episodic 记忆 + 用租约避免冲突。
  • 团队 Onboarding：新成员的 Agent 直接继承团队 semantic + procedural 记忆（架构决策、踩坑经验）。
  • 与 LangChain / AutoGen / CrewAI 集成：通过 REST API（/observe / /remember / /smart-search）作为这些框架的外置记忆后端，任意自定义 Agent 均可接入。
