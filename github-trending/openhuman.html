TL;DR
OpenHuman 把"个人 AI 助理"的产品形态从聊天窗口推到了一个有 Memory Tree 长期记忆、118+ OAuth 集成、能加入 Google Meet 的桌面吉祥物。基于 Tauri + Rust 构建、可对接 Ollama 完全离线运行、用 TokenJuice 把 token 用量压低 80%。本地优先、全开源（GPL-3.0），定位在 OpenClaw、Hermes 之外开辟了第三条引人注目的开源路线。

01项目概述

OpenHuman 来自团队 tinyhumansai，2026 年 5 月在 GitHub Trending 榜首附近停留多日。它的自我定位非常直白——一句口号涵盖三个关键词：

"Your Personal AI super intelligence. Private, Simple and extremely powerful."
— tinyhumansai · OpenHuman README

它要解决的不是"再造一个 ChatGPT"，而是把长期记忆、百级工具调度、语音对话、以及桌面吉祥物式的拟人交互四件事打包成本地优先的桌面应用，使 AI 真正成为"日常工作伙伴"而非云端聊天框。

02核心能力四象限

项目把功能整理成四个相互正交的能力面——桌面体验、记忆与知识库、工具集成、隐私与开源。下图把核心能力按层叠加。

记忆与知识库

  • Memory Tree. 将压缩后的知识结构化存储于 SQLite 本地数据库，是 AI 的"第二大脑"。
  • Obsidian 兼容 Vault. AI 写入的笔记直接落入用户既有的 Obsidian 工作流，浏览编辑用熟悉的工具。
  • 自动同步. 每 20 分钟抓取新数据，保持知识库新鲜，无需手动整理。
  • TokenJuice 智能压缩. 把上下文 token 用量降低最多 80%，显著降低调用成本与延迟。

工具与集成

  • 118+ 第三方 OAuth 集成：Gmail · Notion · GitHub · Slack · Stripe …
  • 内置 Coder 工具集：文件系统 · Git · Lint · Test · Grep。
  • Web 检索与抓取原生 Web Search + Scraping。
  • 语音能力：STT + ElevenLabs TTS + 吉祥物对口型动画。
  • 模型路由：自动把不同类型的任务派发给最合适的 LLM。
  • 本地模型支持：可选 Ollama，完全离线运行。

03技术架构 / 技术栈

Rust 占比近 七成，TypeScript 负责 UI 与业务编排——这是一个"AI 桌面应用 2026 范式"的典型构成：系统级语言守护记忆与工具调度的性能，前端工程驱动可视化与产品迭代。

Tauri 取代 Electron 是这个时代桌面 AI 应用的关键选型——更小的二进制、更紧的内存、更接近原生的体验，与 Rust 的能力刚好匹配。

05与同类项目的差异化

项目方在 README 中把 OpenHuman 与 Claude Cowork、OpenClaw、Hermes Agent 做了横向对比。下表抽取关键差异：

维度 OpenHuman 一般竞品
开源协议 完整开源 GPL-3.0 多为闭源 SaaS 或部分开源
集成数量 118+ OAuth 通常几十个
持久记忆 内置 Memory Tree + Obsidian Vault 多数依赖外部向量库或无记忆
本地模型 原生 Ollama 支持 多为云端模型
桌面形态 Tauri 原生 · 吉祥物 · 视频会议参与 多为聊天窗口
Token 优化 TokenJuice 内置压缩 -80% 通常无

一句话：它把"个人 AI 助理"从聊天窗口推到了"有记忆、有表情、会开会、能跑本地模型的桌面同伴"。

06如何快速上手

macOS / Linux x64

curl -fsSL https://raw.githubusercontent.com/tinyhumansai/openhuman/main/scripts/install.sh | bash

Windows (PowerShell)

irm https://raw.githubusercontent.com/tinyhumansai/openhuman/main/scripts/install.ps1 | iex

从源码开发

# 先确保已安装 Git · Node.js 24+ · pnpm 10.10.0 · Rust 1.93.0 · CMake
pnpm install
pnpm dev                                # 仅启动 Web UI
pnpm --filter openhuman-app dev:app     # 启动桌面 Shell
pnpm typecheck && pnpm format:check && cargo check

首次配置三步

  1. OAuth 授权. 按需勾选 Gmail / Notion / Slack 等第三方。
  2. 选择 LLM 源. 云端 API（OpenAI / Anthropic 等） 或 本地 Ollama。
  3. 指定 Vault 路径. 已有 Obsidian Vault 直接接入；没有则新建。
CAUTION · Early Beta
当前版本仍在快速迭代（v0.53.40，2026-05-13）。生产关键数据建议先在隔离 Vault 中试用，避免不可逆同步事故。
