Worker 调度：能力匹配、隔离工作区与 PromptRunner

这一页看 manager 如何把 DAG envelope 派发给 worker/node，以及 worker 如何把 agent 事件变成 DAG 事件。

Dispatch 优先 worker，再 provision，再 fallback node

WsDispatchAdapter 先查 requiredCapabilities，并拒绝 provision 正在进行的节点；有匹配 worker 就直接派发，没有 worker 但声明了 capabilities 时会硬失败，不会偷跑到 node；没有 capability 要求时才尝试 Docker-capable node provision，最后才 fallback 到普通 node。源码证据是 sources/homerail/homerail_manager/src/orchestration/ws-dispatch-adapter.ts:134、sources/homerail/homerail_manager/src/orchestration/ws-dispatch-adapter.ts:139、sources/homerail/homerail_manager/src/orchestration/ws-dispatch-adapter.ts:144、sources/homerail/homerail_manager/src/orchestration/ws-dispatch-adapter.ts:165、sources/homerail/homerail_manager/src/orchestration/ws-dispatch-adapter.ts:175、sources/homerail/homerail_manager/src/orchestration/ws-dispatch-adapter.ts:189。

隔离工作区避免复用 generic worker

 240      );

 242      if (exactRunScopedWorker) return exactRunScopedWorker;

 251

 253      this.nextWorkerIndex = (this.nextWorkerIndex + 1) % genericWorkers.length;
 254      return worker;
