§ 00 · OVERVIEW

它解决什么，为什么值得拆

FableCut 的独特价值不是“又一个网页剪辑器”，而是把时间线抽象成 agent 可写的 JSON 契约。源码注释直接把目标写在 sources/fablecut/app.js:1 到 sources/fablecut/app.js:6：浏览器可独立运行，也可以连接 server.js 获得持久化 project.json、媒体库和 REST API，因此外部工具能程序化编辑时间线。

这套架构非常适合今天的 coding-agent/dev-tool 主题：UI 负责交互和预览，服务端做文件与编码，MCP 层暴露可控的修改面。项目模型从见 sources/fablecut/app.js:91；运行态缓存从见 sources/fablecut/app.js:100；连接服务端从见 sources/fablecut/app.js:222；乐观保存从见 sources/fablecut/app.js:264；外部同步从见 sources/fablecut/app.js:293。

▸速度曲线与媒体时间

速度不是简单倍速；当存在 keyframe speed ramp 时，mediaTimeAt 用小步积分把时间线时间映射回素材时间。速度积分开始见 sources/fablecut/app.js:186；波形绘制复用该映射见 sources/fablecut/app.js:758；音频混音也读取 speed ramp见 sources/fablecut/app.js:2238；快速导出逐帧 seek见 sources/fablecut/app.js:2197。

sources/fablecut/app.js:186-210JavaScript
186function mediaTimeAt(c, t) {
187  const base = clipSpeed(c);
188  const local = clamp(t - c.start, 0, c.duration);
189  if (!hasSpeedRamp(c)) return c.in + local * base;
190  const key = JSON.stringify(c.keyframes.speed) + "|" + c.duration.toFixed(4) + "|" + base;
191  let e = speedIntCache.get(c.id);
192  if (!e || e.key !== key) {
193    const step = 1 / 120;
194    const n = Math.max(2, Math.ceil(c.duration / step) + 1);
195    const cum = new Float32Array(n);
196    let prev = clamp(kfChannel(c, "speed", 0, base), 0.1, 8);
197    for (let i = 1; i < n; i++) {
198      const lt = Math.min(c.duration, i * step);
199      const cur = clamp(kfChannel(c, "speed", lt, base), 0.1, 8);
200      cum[i] = cum[i - 1] + ((prev + cur) / 2) * (lt - (i - 1) * step);
201      prev = cur;
202    }
203    e = { key, cum, step };
204    speedIntCache.set(c.id, e);
205  }
206  const idx = Math.min(e.cum.length - 1, local / e.step);
207  const i0 = Math.floor(idx), frac = idx - i0;
208  const v = i0 >= e.cum.length - 1 ? e.cum[e.cum.length - 1]
209          : e.cum[i0] + (e.cum[i0 + 1] - e.cum[i0]) * frac;
210  return c.in + v;

▸风险与边界

大文件与内存
项目是浏览器本地合成，长片或高分辨率素材会把压力放在 Canvas、Video seek、AudioBuffer 和 MediaPipe mask 上。
并发写入
revision 能防止 stale overwrite，但没有更细粒度的 merge；多个 agent 同时改同一 clip 仍需要上层协调。
安全面
MCP import_media 接受本地路径并复制到媒体目录，适合本机 agent，不适合无隔离的多租户服务。
工程成熟度
零依赖让代码易读，但单个 app.js 较大；后续若功能继续扩张，模块化会变得更重要。
