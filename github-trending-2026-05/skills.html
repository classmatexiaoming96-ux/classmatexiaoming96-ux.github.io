01项目概述

仓库 mattpocock/skills 的副标题非常坦诚：

"Skills for Real Engineers. Straight from my .claude directory."
— Matt Pocock

它不是又一个 Agent 框架，也不是"Prompt 模板大全"，而是把 40 年软件工程智慧翻译给 AI 看——每个技能背后都有具名著作支撑，可在任意 Agent（首推 Claude Code）中"复制即用"。

0318 个开箱即用技能

所有技能按用途分三组——核心是 10 个工程类，其次是节省 token 的 4 个生产力类，最后是 4 个工具类杂项。

Engineering · 10 个

命令作用
/diagnose结构化 Debug：复现 → 最小化 → 假设 → 注入观察 → 修复
/grill-with-docs用领域语言追问需求并更新 ADR
/triage状态机驱动的 Issue 分诊工作流
/tddRed-Green-Refactor 测试驱动开发
/to-prd把会话上下文转成正式 PRD 文档
/to-issues把规格拆成可独立认领的垂直切片
/improve-codebase-architecture识别并解决架构退化
/zoom-out要求 AI 对陌生代码先建立更广视野
/prototype一次性原型验证设计
/setup-matt-pocock-skills仓库配置脚手架

Productivity · 4 个

  • /caveman. "穴居人模式"，压缩到极简表达，节省 ~75% tokens。
  • /grill-me. 让 AI 反过来彻底拷问你的需求。
  • /handoff. 会话压缩，方便 Agent 间交接。
  • /write-a-skill. 创作新技能的元技能。

Miscellaneous · 4 个

  • /git-guardrails-claude-code. 拦截危险 Git 命令。
  • /migrate-to-shoehorn. 类型断言迁移工具。
  • /scaffold-exercises. 创建教学练习结构。
  • /setup-pre-commit. 配置 Husky 钩子（lint + test）。

04典型工作流

把若干技能串成一条流——从含糊需求到逐行 TDD，全程不脱离一致的"共同语言"。

NOTE · 投资一次 · 受益每次
作者特别强调 CONTEXT.md 模板的价值——一旦项目共同语言建立，后续每次会话都能用极少 token 完成沟通。这是整套技能里 ROI 最高的一项。

05技术架构

维度选型
语言Shell 100%（核心是脚本 + Markdown 模板）
协议MIT
首选客户端Claude Code（但兼容任意支持 skills 模式的 Agent）
关键文件CLAUDE.md（Agent 配置）· CONTEXT.md（项目共同语言文档）
安装器skills.sh / npx skills 命令行工具
与多数 AI 项目动辄数万行 TS/Python 代码不同，本仓库全部"技术含量"在于领域专家把工程方法论翻译为 Agent 能消费的 Prompt。

典型场景

  • 写代码前. 用 /grill-with-docs 把含糊需求逼问清晰。
  • 启动新 feature. 走 /to-prd → /to-issues"规格 → 切片"流程。
  • 调试线上问题. 用 /diagnose 的结构化方法。
  • 长会话. /caveman 节省 token、/handoff 在 Agent 间换班。
  • 定期审计. /improve-codebase-architecture 治理技术债。

08如何快速上手

30 秒安装

npx skills@latest add mattpocock/skills

初始化配置

在 Claude Code 中运行：

/setup-matt-pocock-skills

会依次询问：

  1. Issue 跟踪选择（GitHub / Linear / 本地文件）
  2. 分诊标签定义
  3. 文档目录偏好

从需求到 TDD 的最小示例

# 1. 在写代码前先彻底厘清需求
/grill-with-docs

# 2. 把会话沉淀成 PRD
/to-prd

# 3. 拆成可执行的垂直切片
/to-issues

# 4. 每个切片用 TDD 实现
/tdd

# 5. 期间发现 Bug
/diagnose

# 6. 长会话切换
/handoff
CAUTION · 建议先读 CONTEXT.md 模板
整套技能的复利效应来自项目共同语言。如果跳过 CONTEXT.md 直接跑流程，等于一直在"陌生人模式"下沟通，多数节省 token 的承诺都无法兑现。
