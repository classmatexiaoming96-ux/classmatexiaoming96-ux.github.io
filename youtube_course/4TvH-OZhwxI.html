别再把 AI Agent 当聊天机器人用

AI Agents Explained: How to Create and Use AI Agents in 2026

这期视频的核心不是再解释一遍“AI 很强”，而是把一个更具体的问题摊开：你可能已经在为 Agent 级别的能力付费，却仍然用聊天机器人的方式使用它。页面下方把视频内容整理成中文文章，重点讲清楚 Chatbot 和 Agent 的差别、四类平台怎么选、Prompt Contracts 怎么写，以及为什么记忆文件会让 Agent 越用越像长期同事。

一问题引入：AI 时代最贵的错误

视频开头用了一个很直接的对比：同样是 Claude，同样每月二十美元左右的订阅，有人把它当作聊天窗口，有人却让它打开浏览器、读文件、批量改名、跑命令、整理表格，并在午饭前完成原本要花一周的琐碎工作。

这就是“AI 时代最贵的错误”：你买到的可能已经是第二种工具，却还在用第一种方式提问。聊天机器人可以告诉你“应该如何整理下载文件夹”，Agent 则可以真的进入文件夹，读取几十个 PDF、截图、收据和合同，按日期和供应商重命名，拆成分类目录，再输出一张税务季可用的费用表。

这不是提示词运气好，也不是模型突然变聪明。视频反复强调，Agent 能跑起来，是因为模型之外接上了工具、记忆、目标和循环。只要你仍然把请求写成“帮我想想”“给我一些建议”，它就会倾向于返回文字；当你把请求写成有终点、有边界、有交付格式的任务合同，它才更可能真正把事情做完。

A chatbot can tell you how to do this. An agent just does it. 视频开场对聊天机器人和 Agent 的核心区分

所以这期视频值得看的地方，不在于又列了几个热门工具，而在于它把“如何翻转开关”讲得很具体：先建立概念模型，再选平台，最后用 Prompt Contract 和 Memory File 把 Agent 从一次性问答变成可复用的工作系统。

二核心概念：Chatbot 和 Agent 到底差在哪里

视频最重要的基础判断是：Chatbot 和 Agent 可以运行在完全相同的大模型上，脑子可能是同一个，差异在于脑子周围接了什么。一个世界级厨师坐在空房间里，没有炉子、没有食材、没有订单，他仍然懂烹饪，却做不出菜。Agent 的差异正是这些“周边设施”。

聊天机器人是“只有脑子”的形态。你输入一句话，它返回一句或一段答案，然后对话结束。它通常不能自己打开文件、运行命令、访问浏览器、改动项目，也不会天然记得你上周教过它的偏好。它很有用，但被框在聊天窗口里。

Agent 是同一个脑子外接四个关键组件：LLM、Tools、Memory、Goals。LLM 是推理引擎；Tools 是手，包括终端、浏览器、文件系统、API；Memory 是笔记本，让它每次开始前读到项目约定和历史教训；Goals 是目的地，不是模糊愿望，而是有完成定义的结果。

四个核心组件：LLM 负责判断，Tools 负责行动，Memory 负责延续上下文，Goals 负责定义完成标准。缺任何一个，系统都会退化：没有工具就只能建议，没有记忆就重复犯错，没有目标就容易跑偏。

Observe-Think-Act 循环

把四个组件串起来的是一个简单循环：Observe、Think、Act。Observe 是观察当前世界状态，例如文件是否存在、页面显示什么、上一条命令返回了什么；Think 是基于当前事实决定下一步，而不是死守五分钟前的假设；Act 是调用工具改变某些东西。然后它再次观察，直到结果满足目标。

这个循环解释了为什么 Agent 不应该只被当成“高级聊天框”。真正的 Agent 会在行动后检查结果，会因为命令失败而换路，会因为文件内容与预期不同而调整计划。你给它的不是“一个问题”，而是“一个可验证的任务”。

Observe, think, act, checking against the goal until done. 视频对 Agent 引擎的最短描述

三关键论点与实操：四个平台、两套习惯

视频说市面上有很多 Agent 平台，但作者长期测试后把四个留在自己的工作流里：Claude Code、OpenAI Codex、OpenClaw、Google Antigravity。它们不是互相替代的同一种产品，而是各自擅长不同的工作形态。

1. Claude Code：适合复杂编排和可解释推理

Claude Code 被视频描述为 Anthropic 的官方 Agent，能在本机运行，适合 Mac、Windows 以及 Windows ARM 64。视频特别提醒：免费的 Claude.ai 不包含 Claude Code，需要付费 Anthropic 方案；同一订阅同时覆盖网页聊天和 Claude Code。

它的强项是可观察的推理过程。你能看到模型如何思考、如何调用工具、如何编辑文件，也可以在任务进行中排队追加消息或重新引导。视频还强调，Claude Code 不只是写代码工具。开头的下载文件夹整理、批量重命名、PDF 解析、截图抽取和视频处理任务，都属于它能处理的通用电脑工作。

适合把 Claude Code 当“思考伙伴”而不是“导弹”使用：复杂调试、跨文件重构、项目整理、长链路自动化、需要你中途检查和纠偏的任务，都更适合它。

2. OpenAI Codex：适合工程仓库里的执行型修改

视频把 Codex 放在第二个 Agent 平台讲，重点是 OpenAI 在编码工作流上的能力。对于开发者来说，Codex 的价值不是“会回答代码问题”，而是能在仓库里读上下文、改文件、运行测试、根据失败结果继续修补。

如果你的任务是补测试、改接口、写页面、重构某个模块、准备 pull request，那么 Codex 这类代码型 Agent 更接近“能直接落地的工程协作者”。它需要清晰的文件边界、测试命令和完成标准；你越把它当成仓库里的执行者，而不是百科问答，它的价值越明显。

3. OpenClaw：适合跨应用的个人生产力自动化

OpenClaw 在视频里更偏向应用层 Agent：网页仪表盘、聊天界面、市场集成，以及和 Telegram Bot 组合使用。它适合把 Notion、Google Docs、演示文稿、任务管理和消息通知串起来，让 Agent 在多个日常工具之间搬运信息、总结内容、分发任务。

这类平台的关键不在代码能力，而在连接能力。你要自动整理信息流、把想法变成看板、把文档摘要推到消息软件，OpenClaw 这类产品会比纯代码 Agent 更贴近日常工作。

4. Google Antigravity：适合 Google 生态和长上下文工作

视频把 Google Antigravity 作为 Google/Gemini 阵营的 Agent 形态介绍。它的优势通常来自 Google Workspace、Gemini 长上下文以及与 Docs、Sheets、Gmail、Drive 等资料源的结合。

如果你的工作主要沉淀在 Google 文档、表格、邮件和云盘里，Antigravity 代表的是另一种入口：不是先进入代码仓库，而是先进入工作资料库。它更适合跨文档归纳、表格整理、邮件上下文分析和基于 Workspace 的任务推进。

Prompt Contracts：从描述愿望到签任务合同

视频后半段最实用的部分是 Prompt Contract。作者认为，修复 Agent 跑偏的答案不是“写更长的提示词”，而是写更结构化的提示词。一个真实的 Agent 请求应该包含四段：Goal、Constraints、Format、Failure。

Goal

写结果，不写动作。不是“做一个落地页”，而是“做一个单页产品发布站，首屏推动用户留下邮箱，并可在本地预览”。

Constraints

写护栏。不能装依赖、不能碰某个目录、不能部署、不能用外部 CDN、不能复制竞品文案，都应提前说明。

Format

写交付物形状。单个 index.html、一个目录、JSON schema、Markdown 报告、补丁 diff，都要在开始前定清楚。

Failure

写卡住时怎么办。缺关键信息就停下来问；工具连续失败两次就报告尝试过程；允许猜测时也要标注假设。

这四段之所以有效，是因为它们分别解决了 Agent 执行中的四类不确定性：不知道完成标准、不知道边界、不知道交付格式、不知道失败策略。没有这些信息时，Agent 会自己发明答案，而且这个发明过程会消耗真实 token 和真实时间。

Memory 文件：让 Agent 不要每次重新入职

视频最后一段讲 Memory。每个严肃平台都有类似能力，只是文件名不同：Claude Code 读 CLAUDE.md，OpenClaw 可以读 AGENTS.md 一类项目规则，Antigravity 也有自己的记忆约定。共同点是：项目根目录里放一个纯文本规则文件，Agent 每次开工前都会读。

作者给了一个很具体的例子：他做 B2B 客户仪表盘时，Agent 每次都往面向客户的文案里塞表情符号。手工删一次，下次又回来。后来他让 Agent 在项目根目录创建 CLAUDE.md，写入“除非明确要求，不要在客户可见文案里使用表情符号；这是 B2B 产品”。这个问题之后就消失了。

更高级的技巧是让记忆文件自我更新：每次任务结束前，如果用户纠正了你，或者你因为错误假设踩坑，就把新的规则追加到文件底部的 learned rules 区域。这样第一轮只有一条规则，第五轮可能有二十条，第二十轮时 Agent 已经积累了足够多的项目偏好，不会反复犯同一种错。

五引用块：视频里的关键金句

这些句子可以当成使用 Agent 的操作原则。英文保留原意，中文解释放在后面，方便复盘时直接引用。

Most people are paying for the second one and using it like the first. 多数人付的是 Agent 的钱，却按聊天机器人的方式使用。
The fix isn't a longer prompt, it's a structured one. Agent 跑偏时，优先修结构，而不是把提示词越写越长。
Goal, constraints, format, failure. Memorize those four words. 每个严肃 Agent 任务都应该回答这四件事。
Description versus contract, that's the whole shift. 从“描述需求”转向“签订任务合同”，是使用方式的根本变化。
The agent isn't broken. You just never told it to remember. 很多重复错误不是模型坏了，而是你没有把规则写进记忆。
Stop reading about AI agents. Start running them. 最终行动建议：选一个真实任务，从今晚开始跑完整链路。
