二、Fable 5 的两大核心能力

[10:56]单次正确性（Single-shot Correctness）：给 Fable 5 一个复杂的、描述清晰的问题，它往往在第一次就能做对。早期的测试者反馈，曾有单个 prompt 完成了过去需要团队花数天甚至数周才能完成的工作。

[11:23]长时程自主性（Long Horizon Autonomy）：Fable 5 可以在一个目标上连续运行数天仍保持连贯性。它能记住跨越数百万 token 上下文窗口的规格说明，并能调度子 Agent、保持它们在轨道上运行——比其他模型更可靠、更具成本意识。

[12:33]Fable 5 不仅擅长写代码，更擅长读代码：它能梳理代码库排查线上故障、挖掘 git 历史定位问题根源，还能主动提出改进建议。同时它也能处理财务报表、文档、幻灯片等"企业实际运行的工作"，端到端完成。

五、AI 原生公司：三个关键要素

[21:08]Angela Jen 描述了一个"AI 原生公司"的场景：某款产品夜里自动发现了自己的错误，读取错误报告、找到 bug、写出修复方案并向所有用户推送——团队醒来时问题已消失，连 changelog 都写好了。没有 standup，没有 ticket，这是 AI 原生公司的样子。

[21:51]实现这一愿景需要三个关键要素：Harness（工具链）、Context（上下文）、Infrastructure（基础设施）。Claude Managed Agents 正是将三者打包在一起的产品化方案。

Fable 5 是构建长时程 Agent 的最佳模型，Managed Agents 是专为 Claude 打造的 Agent 运行平台——二者结合，可以用显著更少的努力获得更好的 Agent 成果。

六、Cloud Managed Agents 核心功能

[22:19]Harness（工具链）：赋予 Claude 模型实际执行工作的能力，包括工具、环境和行动权限。Managed Agents 将"大脑"（决策）和"手"（执行/沙箱）分离。

[23:05]Context（上下文）：提供 100 万 token 上下文窗口、Agent 记忆系统（Memory）、Agent 自主读写技能（Skills）以及"做梦"（Dreaming）能力——Agent 可以回顾自身的历史轨迹并识别自我改进方向。

[23:58]Infrastructure（基础设施）：构建长时间运行的自主任 Agent 需要极致的规模和可靠性。Managed Agents 自动扩缩容沙箱，并能生成多 Agent 舰队来保障可靠性和持久性。

[26:44]支持定时部署（schedule deployments）和Vault（环境变量安全存储，Agent 可在不接触密钥的情况下进行经过身份验证的 API 调用）。

七、Dreaming：Agent 的自我反思能力

[28:51]"做梦"（Dreaming）功能让 Agent 可以回顾所有历史会话，将发现写入记忆和技能，以便下次表现得更好。这是让 Agent 真正实现自我改进的关键特性。

在 Shankiro Racing 团队的演示中：四个研究项目（空气动力学、轮胎温度、动力单元、驾驶安全）各自由一个 Claude Managed Agent 负责，Agent 可以根据预定义的评分标准（rubric）自主迭代，直到输出达到"好"的标准。

八、Claude Code 最新进展

[30:41]Claude Code 的使命是弥合"想法"与"已发布产品"之间的鸿沟。借助 Claude Frontier 智能，让每个构建者都能用上最强大的 AI 能力。

[32:11]Claude Code 从 CLI 起步，逐步扩展到 IDE（图形化跟进所有代码变更），再到多会话并行（multi-clotting），以及最新的两个界面：Cloud Desktop（全屏图形界面，内置预览、侧边控制台）和 Agent View in CLI（不离开终端也能看到所有 Agent 的等待/运行/完成状态）。

[33:42]在 Anthropic 内部，工程师平均代码产出是去年的 8 倍，与此同时工程团队规模还在显著增长。

企业大规模采用：Spotify 用 Claude 迁移数千个代码库，每月合并超过 1000 个 PR，迁移时间减少 90%；日本 Mercari 整个工程团队跑在 Claude Code 上，产出同比提升 90%。
