Microsoft Agent Framework

⏱ 60 分钟 📺 Microsoft Agent Framework 🧑‍💻 入门实战

一课程定位

这是一门面向初学者的 Microsoft Agent Framework 实战课。视频先解释 LLM、聊天机器人和 AI Agent 的差异，再进入环境搭建、Hello World Agent、记忆、上下文、工具调用和函数工具。

作者反复强调：Agent 不只是“输入问题、返回答案”的聊天框，而是一个具备推理、上下文、记忆和外部行动能力的系统。

二Agent 的基本模型

课程把 Agent 拆成三个部分：脑、记忆和手。脑负责根据上下文推理，记忆保存对话线程或用户偏好，手则通过工具、函数调用、API 或 MCP 与外部世界交互。

组成作用视频中的例子
上下文当前任务需要知道的知识和约束系统提示、文件、数据库、业务规则
记忆跨轮次保留有价值的信息用户姓名、出发城市、历史偏好
工具连接外部系统并执行动作查航班、订票、查询用户数据库

三框架提供了什么

Microsoft Agent Framework 的价值在于把模型、记忆、上下文管理、工具调用和编排收拢到统一结构里。课程里提到的 Agent 类型包括通用 AI Agent、面向对话流程的 Chat Agent，以及和 Azure AI 服务结合更紧的 Azure AI Agent。

如果把普通聊天机器人看成会回答问题的脑，那么 Agent Framework 试图给它加上记忆、工具和可编排的执行流程。

四实战路径

  1. 环境搭建：创建 Python 虚拟环境，安装 Agent Framework 相关包，配置模型和 API key。
  2. Hello World：先跑一个最小 Agent，验证模型连接、提示词和响应链路。
  3. 航班规划 Agent：通过“John Doe 从巴黎去伦敦”的例子说明上下文和记忆为什么重要。
  4. 函数工具：把外部函数注册成 Agent 可调用工具，让 Agent 能查数据、补信息、返回结构化结果。
  5. 测试任务：用一组用例验证工具调用、响应格式和异常路径是否符合预期。

五记忆与上下文的取舍

课程中特别区分了上下文和记忆：上下文是本轮任务需要的材料，记忆是系统选择长期或短期保存的内容。并不是所有信息都应该写入记忆，过多记忆会污染判断，也会增加成本。

实践建议是：用户偏好、身份信息和需要跨会话复用的事实可以进入记忆；临时推理步骤、一次性工具结果和已经完成的中间状态更适合留在当前上下文里。

六函数工具的关键点

后半段课程围绕函数工具展开：把普通函数声明为 Agent 可用的工具，明确输入参数、返回格式和错误处理。示例里还要求指定某个工具，例如 query_user_database，并用测试用例确认有效用户查询可以成功。

  • 工具描述要清楚，否则模型不知道何时调用。
  • 返回值最好结构化，便于后续步骤消费。
  • 工具调用必须可测试，尤其是失败路径和格式约束。
