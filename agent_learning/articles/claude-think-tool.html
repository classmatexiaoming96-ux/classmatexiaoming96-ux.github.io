"think" 工具：让 Claude 能停下来思考

原文：The "think" tool: Enabling Claude to stop and think

导读

"think" tool 是一个结构化 scratchpad，让 Claude 在复杂 tool-use workflow 中停下来分析。它不获取新信息、不改变外部状态，只帮助模型整理中途获得的信息。

核心观点

  • "think" tool 与 extended thinking 不同，它发生在工具调用链中途。
  • 它适合长工具链、多轮对话、政策密集环境和代价高的顺序决策。
  • 工具本身无副作用，只记录结构化推理。
  • 在 tau-bench airline 任务中，配合优化提示显著提升表现。
  • 最佳提示要求列出规则、检查必要信息、验证政策合规并审视工具结果。
  • 简单指令遵循和并行工具调用通常不需要 think tool。

正文解读

为什么需要中途思考

复杂工具任务的难点常发生在工具返回之后。模型需要把新信息与规则、用户目标和下一步动作对齐，而不是机械继续调用工具。

"think" tool 给了模型一个低风险空间，让它在行动前整理政策、检查缺失信息和评估工具结果。

适合与不适合

适合场景包括客服政策、航空改签、零售退货、长链路工具调用和顺序决策。每一步都可能改变后续选择，因此中途反思有价值。

不适合场景则是简单指令、单次工具调用或互不依赖的并行工具调用。此时额外 scratchpad 只会增加成本。

与 extended thinking 的关系

文章后来更新说明，多数情况下现代 extended thinking 更推荐。它能覆盖许多需要深度推理的场景。

但 think tool 作为设计模式仍有启发：当工具结果改变任务状态时，系统可以提供一个无副作用、可审计的中途推理节点。

深度解读

这篇文章的价值不只在某个工具本身，而在一个 agent 设计模式：让模型在行动前有结构化停顿。工具调用系统容易鼓励模型连续行动，而高风险流程需要先核对规则、证据和缺口。无副作用 scratchpad 是一种低成本护栏，让 agent 的下一步更可解释。

落地启示

  • 只在复杂顺序决策和政策密集流程中引入中途思考。
  • 让 scratchpad 明确检查规则、缺失信息和工具结果。
  • 不要让 think tool 获取信息或改变状态。
  • 用 eval 判断它是否真的提升成功率。
  • 能用 extended thinking 覆盖时，优先使用更现代的内建能力。
